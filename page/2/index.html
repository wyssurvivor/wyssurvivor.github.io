HA-AND-REPLICATION

发表于 2015-11-24   |   分类于 distributed system

ha 和 replication

  • 强同步：master在收到commit之后，当所有slave都写成功之后，才会返回成功。
  • 半同步：当有一定数量的slave返回之后，master就会返回成功。
  • 异步：master收到commit，在本地写成功之后，直接返回成功。本地会有线程给slave发消息，slave有线程接受消息，并执行。

mysql 的可靠性解决方案

mysql默认使用的是异步的同步方式，在mysql5.6中加入了一种半同步方式，当有一半的slave返回时，master就返回成功。在5.7中加入了一个lossless模式，主要是解决半同步方式中的数据丢失的情况，我没有仔细看过具体的实现方式，等以后看过再更新。

在我刚毕业来公司接的第一个任务就涉及到了mysql的可靠性方案的问题。在我的代码中有这样一段逻辑：像mysql中插入数据然后马上取出表中所有的数据。当时就出现了非常奇怪的现象，当我得到取出来的数据时，发现之前刚刚执行完的插入操作并没生效。当时，我的mentor告诉我是master的数据还没有同步到slave上，而读操作时从slave上读取的。现在想来，在当时测试环境下的mysql配置使用的是异步方式同步数据。

接下来，可能会更新mysql replcation的部署实例。

一键部署到tomcat

在写完业务逻辑代码之后，可以开始着手部署了。

apache-tomcat 配置conf/tomcat-users.xml。在里面加入tomcat manager访问权限。此处注意不要写在注释里面了，我第一次搞的时候没注意，写在注释里面了，结果tomcat manager一直登陆不上去，浪费了很多时间。

<tomcat-users>


<role rolename="admin-gui"/>
<role rolename="admin-script"/>
<role rolename="manager-gui"/>
<role rolename="manager-script"/>
<role rolename="manager-jmx"/>
<role rolename="manager-status"/>
<user username="admin" password="admin" roles="manager-gui,manager-script,manager-jmx,manager-status,admin-script,admin-gui"/>

</tomcat-users>

maven配置，让能够访问tomcat，这个地方应该是需要权限。

<servers>

<!-- 配置tomcat-/manager/text 访问权限 -->
<server>
  <id>tomcat7</id>
  <username>admin</username>
  <password>admin</password>
</server>

</servers>

pom.xml 加入build,并配置tomcat7的maven插件

<build>
    <finalName>myApp</finalName>
    <!-- directory缺省情况下指向target --> 
    <!--<directory>${basedir}/target</directory>-->
    <plugins>
        <plugin>
            <groupId>org.apache.tomcat.maven</groupId>
            <artifactId>tomcat7-maven-plugin</artifactId>
            <version>2.2</version>
            <configuration>
                <url>http://localhost:8080/manager/text</url>
                <!-- server、username、password对应maven的setting下的配置 -->
                <server>tomcat7</server>
                <username>admin</username>
                <password>admin</password>
                <path>/${project.build.finalName}</path>
                <!-- war文件路径缺省情况下指向target -->
                <!--<warFile>${basedir}/target/${project.build.finalName}.war</warFile>-->
            </configuration>
        </plugin>
    </plugins>
</build>

以上基本完成了部署配置，可以开始部署了

  • mvn clean install 清理并编译到本地库中
  • mvn tomcat7:redeploy 第一次发布用tomcat7:deploy,再次发布用tomcat7:redeploy

然后可以在浏览器中通过url访问你的业务逻辑了
